## 🎯 Project Overview

**Namka Control** is a centralized command hub for managing Ali Mora's portfolio of AI-assisted SaaS projects. It tracks priorities, blockers, AI agent assignments, and infrastructure across all active repositories.

### Core Philosophy
- **Master.md as Source of Truth**: The dashboard does not use a traditional database. It fetches and parses `Master.md` directly from GitHub.
- **MACP Workflow**: Implementing the Multi-Agent Coordination Protocol where agents (like you, Gemini, Antigravity, and Comet) collaborate, cross-check, and build consensus.
- **High Velocity**: Building with Next.js 15, shadcn/ui, and Tailwind CSS for rapid deployment.

## 🏗 Starting Architecture

### Technical Stack
- **Framework**: Next.js 15 (App Router)
- **Language**: TypeScript
- **Styling**: Tailwind CSS
- **UI Components**: shadcn/ui
- **Data Fetching**: GitHub REST API (Octokit or native fetch)
- **State Management**: React Context / Hooks

### Component Architecture

1. **Layout Shell**
   - Navigation Sidebar (Project list, Agent status)
   - Header (Global status, Thermal metrics)
   - Command Center (Terminal-style input for MACP)

2. **Dashboard Views**
   - **Project Grid**: Dynamic project cards parsed from Master.md
   - **Agent Activity**: Feed showing latest review entries from all agents
   - **Status Badges**: Real-time priority and health indicators

3. **Data Layer (`src/lib/github.ts`)**
   - `fetchMasterMd()`: Raw markdown retrieval
   - `parseMasterMd()`: Logic to extract projects, agents, and reviews
   - `syncMasterMd()`: Webhook integration for instant updates

## 🛠 Your Initial Task

Your first objective is to design the **Markdown Parser Architecture**. Since `Master.md` is our database, we need a robust way to convert structured markdown into JSON objects for the frontend.

### Parsing Requirements
- **Projects**: Extract Name, Repo, Stack, Status, Next Step, AI Model, and Notes.
- **Reviews**: Extract Date, Agent, Status, Project, Topic, Analysis, and Recommendations.
- **Priority Sections**: Correctly categorize projects into Priority 1, 2, or 3 based on markdown headers.
